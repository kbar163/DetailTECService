Crear un endpoint para permitir al usuario agregar, editar o eliminar un tipo de lavado.
Los endpoints reciben la siguiente información en formato JSON al crear y actualizar un tipo de lavado:
{
 “nombre_lavado”: “Lavado y aspirado”,
 “costo_personal”: 2000,
 ”precio”: 4000,
 “duracion”: 60, (son minutos) 
 “puntos otorgados”: 1,
 "costo_puntos": 5,
 "insumos": ["cera", "jabon", "perfume"], (actualiza o agrega a tabla LavadoInsumo)
 "roles": [ { id_rol: 1, tipo_rol: "Lavador" }, { id_rol: 2, tipo_rol: "Aspirador" }] (actualiza o agrega a tabla LavadoRol)
}
Para insumos y roles, insertar o editar sigue este proceso (ejemplo con tabla LavadoInsumo): Se comparan los insumos que se reciben en la lista con los que tiene un lavado asignados en la tabla, los que estén en la lista pero no en la tabla se agregan, los que no están en la lista pero si en la tabla se eliminan. Igual funciona para los roles.
Con esta información se debe retornar los siguientes objetos:
{ "actualizado": true, "mensaje": "Tipo de lavado creado exitosamente" } (ejemplo en caso de que un tipo de lavado se haya creado).

{ "actualizado": false, "mensaje": "Error al crear tipo de lavado" } (ejemplo en caso de error al crear un tipo de lavado).

{ "actualizado": true, "mensaje": "Tipo de lavado actualizada exitosamente" } (ejemplo en caso de que un tipo de lavado se haya actualizado).

{ "actualizado": false, "mensaje": "Error al actualizar tipo de lavado" } (ejemplo en caso de error al actualizar un tipo de lavado).

-El endpoint recibe la siguiente información en formato JSON al eliminar un tipo de lavado: { "nombre_lavado": "Lavado y aspirado"}. Debe eliminar los registros de este lavado en las tablas LavadoInsumo y LavadoRol también. Y con esa información se debe retornar los siguientes objetos:

{ "actualizado": true, "mensaje": "Tipo de lavado eliminado exitosamente" } (ejemplo en caso de que un tipo de lavado se haya eliminado).

{ "actualizado": false, "mensaje": "Error al eliminar tipo de lavado" } (ejemplo en caso de error al eliminar un tipo de lavado).

DoD: Se retorna el valor correcto en el objeto dependiendo del endpoint llamado a través de una herramienta como Postman.
